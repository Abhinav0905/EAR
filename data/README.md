# EAR Evaluation Dataset — Release

Per-question evaluation records for the study *Embedded Agentic Retrieval (EAR): Answer-Quality
Parity With Multi-Call Agentic RAG at a Single Generator Call*. Every answer-quality score is from
an LLM judge (Claude Haiku 4.5, 9 rubrics, run 3× at temperature 0, mean reported) unless noted;
QASPER additionally has token-level Answer-F1 computed separately. All benchmarked generation is
Claude Sonnet 4.5 except the cross-family run (OpenAI GPT-4o-mini) and the mislabel-corrected repeat
runs (see conventions below).

## Record schema

**Primary + repeat + cross-family-generation checkpoints** (`*_checkpoint.jsonl`, `*_repeat.jsonl`):
```
key              "<variant>|<qid>"
corpus           corpus id
variant          pipeline: simple | reranked_simple | agentic | self_rag | flare | ear_full
                 | ablations: ear_rank16 | ear_noadapter | ear_norewrite | ear_norerank
                   | ear_nocoverage | ear_tau03 | ear_tau05 | ear_tau09
gen_model        generation model that produced `answer` (see legend below)
qid, question, category, answerable, reference_answer
metrics          {llm_calls, slm_calls, gen_cost_usd, tokens, wall_ms, iterations}
                 llm_calls = billed generator invocations (the efficiency metric); slm_calls = free local Flan-T5 inferences
judge_cost       USD for the 3-run judge on this record
retrieval_avg, retrieval_avg_std, generation_avg, generation_avg_std, overall
                 axis means over 3 judge passes (1–5); overall = (retrieval_avg+generation_avg)/2
rubric_means     per-rubric means (4 retrieval + 5 generation rubrics)
answer           system answer (truncated to 1500 chars)
retrieved_trim   top-6 retrieved chunks [{page, text(≤300 chars)}]
run_type         (repeat files only) provenance marker; see conventions
```

**`multidoc_checkpoint.jsonl`** adds `src` (source document of the question) and
`top_src_correct` (was the top retrieved chunk from the correct source document); no `rubric_means`.
**`wmp_adaptertest_checkpoint.jsonl`** uses `variant` ∈ {matched, mismatched, noadapter}.

**`crossjudge_gpt4omini.jsonl`**: `{key, corpus, variant, qid, gen_gpt, ret_gpt, overall_gpt,
gen_haiku, ret_haiku, overall_haiku, judge_cost}` — `*_gpt` are GPT-4o-mini judge axis means,
`*_haiku` are the primary Haiku judge's, for the same record.
**`crossjudge_human_gpt4omini.jsonl`**: `{sample_id, corpus, gpt_gen, gpt_overall, had_context,
human1, human2, haiku_overall, sonnet_overall, judge_cost}` — `human1`/`human2` are the two blinded
annotators' 1–5 scores.
**`crossjudge_gpt4omini.json`**: `{model, n_records, corpora, agreement, human, total_cost_usd}`.
**`multidoc_retrieval.json`**: `{docs, top_n, n_questions, by_src_count, retrievers}`; `retrievers`
holds dense / reranked / ear, each with per-source and overall `{top1, recall5, prec5}`.

## `gen_model` legend

| value | model |
|---|---|
| `us.anthropic.claude-sonnet-4-5-20250929-v1:0` | Claude Sonnet 4.5 (primary benchmarked generator) |
| `gpt-4o-mini` | OpenAI GPT-4o-mini (cross-family generation run) |

The judge for every `*_avg` / `*_overall` field in the checkpoints is Claude Haiku 4.5 (the
`crossjudge_*` files hold the GPT-4o-mini judge separately).

## Conventions

**`run_type`.** Present only on the repeat files, value `"sonnet_repeat_temp0"`. It marks records
that are **Sonnet temperature-0 reproducibility re-runs** — the *same* pipelines and generator
(Sonnet) as the primary corpus checkpoints, re-executed to measure temperature-0 reproducibility.
They are **not** a separate model condition and must **not** be pooled with the primary checkpoints
(doing so would double-count). Tooling that aggregates the primary results skips any record whose
`run_type` starts with `sonnet_repeat`.

**Repeat runs.** `wmp_repeat.jsonl` and `arxiv_repeat.jsonl` are the reproducibility re-runs
described above. Their `gen_model` is the Sonnet id (their true generator).

**Generator-swap correction** *(verbatim from results.md §7.3)*:

> **Correction:** the previously reported "Haiku generator swap" was invalid — a harness bug left
> the generator as Sonnet while labeling records Haiku. Those runs are retained honestly as Sonnet
> temp-0 reproducibility re-runs (`wmp_repeat.jsonl`, `arxiv_repeat.jsonl`, `gen_model` corrected +
> `run_type: sonnet_repeat_temp0`); the bug is fixed in the harness.

The genuine, verified generator swap is the cross-family run
(`wmp_gpt4ominiGen_checkpoint.jsonl`): on WMP, EAR ties all five baselines on the generation axis
(Wilcoxon p 0.22–0.96; TOST 5/5 equivalent at δ=0.5 and δ=0.3) at one generator call.
